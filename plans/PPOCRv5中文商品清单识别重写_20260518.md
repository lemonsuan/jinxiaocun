## 目标架构

`Flutter UI -> MethodChannel recognizeTable(imagePath) -> MainActivity -> OcrService(:ocr process) -> Kotlin LiteRT OcrEngine -> JNI C++ PP-OCRv5 -> rows/rawText -> Dart PpStructurePostProcessor -> 可编辑入库草稿`

## 实现边界

* 删除或停用 Android 端 Paddle Lite `.nb` OCR 运行链。
* 引入参考 Demo 的 LiteRT/TFLite PP-OCRv5 引擎、模型和 arm64 native runtime。
* `recognizeTable` 返回结构保持为 `{ rows: List<List<String>>, rawText: String }`。
* OCR 失败、超时或 native 崩溃时，Flutter 主流程仍返回可编辑空草稿。
* 表格结构不依赖 SLANet；先用 OCR 坐标排序和 Dart 后处理解析产品编号、产品名称、数量。
* 保留图片原件保存、人工确认入库、库存流水和出货查询。

## 验收标准

* Android 端不再加载 `libpaddle_light_api_shared.so` 作为 OCR 运行时。
* Android 端能加载 PP-OCRv5 LiteRT 模型并对商品清单图片输出中文 rawText。
* `商品清单.jpg` 或等价商品清单样例能解析出产品编号、中文商品名称和数量草稿。
* Flutter `pp_structure_post_processor_test.dart` 覆盖中文商品清单 OCR 行重建。
* `flutter test` 通过。
* Android debug 或 release APK 能构建成功。
